As the database is used it accumulates debt such as a page having enough logical space to insert new data but fragmentation has caused it to be unusable because it cannot fit into those gaps.
There are background processes (garbage collection) that run to keep the database efficient and small.
# Fragmentation caused by Updates and Deletes
Deleting an entry only deletes the reference to it in the header which makes it unreachable / invisible making that space potentially reusable. Over time this may leave fragmented free space that cannot be reused and thus bloating the db with additional pages.
> [!note]
> In MVCC cells remain until other transactions finish using them. Some systems use a tombstone marker to indicate deletion while others use metadata. Marked cells are invisible to new calls while the previous calls finish using them.
# Page Defragmentation
The process of reclaiming space (garbage collection) and rewriting their contents is called compaction / vaccum.
