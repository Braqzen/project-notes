A database is a collection of subsystems where each subsystem has chosen tradeoffs to optimise for.
Different tradeoffs lead to different architectures and thus a database should be chosen to match the use case or there may be performance issues.
The architecture broadly consists of 3 components which themselves are made of subsystems.
**Query Processor**
The processor consists of 2 systems: Parser and Optimizer.
The Parser takes a query, parses it into an internal representation and validates it.
The Optimizer removes redundant or impossible parts of a query and builds various execution plans for how to best execute the query. This is done through internal statistics of the state of the database.
**Execution Engine**
The execution engine handles the execution of the query and gathers the results from its local node and potentially external nodes.
**Storage Engine**
The execution occurs through the storage engine which handles the data in its memory or on disk.
It consists of multiple components
- Access methods: CRUD API for data
- Lock Manager: handles locks on database object to ensure data integrity
- Transaction Manager: schedules and manages execution and isolation of transactions
- Buffer Manager: caches pages to reduce I/O and increase performance
- Recovery Manager: maintains an operation log and handles system restoration upon inconsistency or failure
> [!note]
> This book is old and systems have evolved but this content is broadly true
