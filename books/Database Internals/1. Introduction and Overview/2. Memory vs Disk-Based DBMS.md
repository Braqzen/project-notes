Memory based databases are significantly faster to access than disk and are easier to use than the (disk) OS abstracted structures.
The downside is potential lack of pesistence in case of a fault when the writing back to disk has not yet occured.
# Durability in Memory-Based Stores
Logs tend to be batched and stored in a log file asynchronously to provide persistence.
Snapshots are taken periodically to prune old state and decrease startup time. If a crash occurs we load the snapshot and replay remaining logs to catch up to the latest state.
In-memory structures have a larger choice of data structures to use than specialised, fixed-size, disk based structures and thus can perform additional optimisations.
