DBMS organise data to store it efficiently on disk, enable efficient access and updates.
A database separates data from indexes by storing them in different files and each table is a separate data file.
## Data Files
Are organised in 3 structures
**Index Tables**
Data is stored in the index.
**Heap Tables**
No requirement for ordering and usually placed in write order. They require additional index structures to become searchable.
**Hash Tables**
Stored in buckets where the hashed key determines the bucket. They're stored in append order or sorted by key.
## Index Files
Maps keys to locations in data files for efficient retrieval.
If the data is stored in the same order as the index key then the index is said to be clustered. Clustered records enable sequential scans which are faster than jumping to other locations to find each record.
## Primary Index as Indirection
Indexes must locate records. They can either store the record's physical location or use an additional level of indirection through another index. Direct references improve reads but require more index updates when records move. Indirection reduces the cost of maintaining indexes when records move but adds an extra lookup during reads.
