Databases are described in terms of a logical model such as a table. On disk, the data is stored in pages where the format is typically organized as entire rows next to each other or columns.  
## Row-Oriented Data Layout
A record consists of multiple columns to represent a row. Rows are stored after each other.  
## Column-Oriented Data Layout
Data is stored in columns so many values from the same column are stored contigiously in the same page. Since data is stored as columns we must either store additional metadata to associate each entry with the other columns or use their stored position as a virtual ID.  
## Distinctions and Optimisations
Reading values of the same column sequentially improves efficiency because it increases cache utilization, enables vectorized CPU instructions, and improves compression by grouping similar data types. If data is accessed as full records with most or all columns needed, row-oriented storage is more suitable.  
## Wide Column Stores
Wide-column stores are different from column-oriented databases. They store data as a multidimensional map where related columns are grouped into column families. On disk, each column family is stored separately, but values belonging to the same row within a column family are stored together.
