Traits that enable cross-thread use
- Send: Ownership can be transferred to another thread
- Sync: Can be shared across threads i.e. if &T is Send
All primitive types are Send + Sync.
These traits are auto traits meaning the compiler implicitly applies them to types when it can, and if a struct consists of types that are all Send + Sync then the whole struct is also Send + Sync.
To opt out add a PhantomData<T> to your type which is zero-sized (takes no space) and doesn't exist at runtime.
To opt in you may implement `unsafe impl Send/Sync for X {}` but it's unsafe because the compiler cannot check correctness.
