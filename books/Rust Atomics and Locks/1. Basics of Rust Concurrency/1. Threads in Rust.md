Processes are isolated from each other however a process can spawn many threads which utilise the same process memory and are thus not isolated between themselves.
A process starts with 1 thread which executes the main function (which may spawn more threads).
When the main thread terminates the program exits even if the other threads haven't finished executing.
You can use the builder to configure the thread such as set a thread name which is used in panic messages and useful for monitoring tools.
