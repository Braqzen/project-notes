Threads that share data cannot modify it but there are types that enable interior mutability.  
Canonical terms are shared and exclusive references rather than mutable and immutable because of the exisence of interior mutability in some types.  
Interior mutability applies to shared references because the rules are slightly bent while exclusive references maintain the same rules.  
# Cell
Limited to single thread use, it allows copying a value or replacing it.  
Allows mutations (exposes an API which uses unsafe blocks) through a shared reference to the wrapper type and not the internal type.  
# RefCell
Limited to single thread use, allows borrowing of contents and tracks with an internal counter.  
Depending on method usage it can panic when borrowing after a mutable borrow occurred.  
# Mutex and RwLock
A RwLock is similar to a RefCell that doesn't panic on borrows because it puts the current thread to sleep when there are conflicting borrows until they resolve.  
A Mutex is similar to a RwLock but it only tracks exclusive borrows.  
# Atomics
An atomic is a concurrent version of Cell forcing us to copy the value instead of borrowing. They do not permit arbitrary types because they have fixed sizes.  
# UnsafeCell
UnsafeCell is a Cell without restrictions and often the underlying wrapper type used for interior mutability by other types and their safe apis.
