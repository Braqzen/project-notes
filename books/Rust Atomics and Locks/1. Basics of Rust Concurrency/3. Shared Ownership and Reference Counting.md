If threads want to share data then neither of them can own it. Something else must be the owner and the data must live as long as the longest running thread.  
# Statics
Statics are owned by the program itself and exist before the main function is run in the main thread.  
# Leaking
You may Box a value and then Box::leak it i.e. release ownership. The box will be static until program termination and thus threads can reference it.  
> [!warning]
> You're leaking memory. Doing this repeatedly will consume all your memory
# Reference Counting
Use an Rc to create 1 allocation of data which others can clone and thus reference. This cannot be sent between threads therefore must upgrade to an Arc.
