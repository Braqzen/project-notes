# Thread Parking
A thread can be put to sleep so it does not consume CPU while waiting to be woken up later.
# Condition Variables
Threads can be woken up conditionally through the use of `Condvar`. A `Convdar` takes a `MutexGuard`, unlocks it and makes the current thread go to sleep atomically to avoid race conditions.
Another thread can acquire the mutex, modify shared state, and then call `notify_one` or `notify_all` on the `Condvar` to wake sleeping threads.
When a thread wakes up, it re-locks the `Mutex` before returning from the wait and continues execution.
